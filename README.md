# gongwen-draft

![gongwen-draft Word 草稿预览](assets/readme-banner.png)

gongwen-draft 是一个中文公文与政务材料写作 Agent skill，帮助机关工作人员起草、改写、审核公文初稿并导出 Word 草稿。

它不是一个独立的软件，而是一套可安装到各类 AI Agent 中的专业技能。安装后，你只需用日常语言描述需求，Agent 就能按照公文规范辅助你完成写作任务。

## 快速上手

**第一步：安装到你的 Agent 中。** 将本项目安装到你所使用的 Agent 或 AI 编程工具的技能目录，刷新后即可使用。具体安装方式取决于你使用的工具，请参考对应工具的文档。

**第二步：直接对话。** 安装完成后，在对话中说明你的需求即可。以下是几个常见场景：

把会议要点整理成一份通知初稿。

把这份请示草稿改写成正式行文，注意行文关系和格式。

帮我审核这份报告的文种、标点、层级序号有没有问题。

把定稿导出为 Word 文件，按公文格式排版。

帮我把这几份素材先整理成事实台账，再起草一份情况专报。

帮我润色这份讲话稿，去掉空泛表述，保留原有事实和口径。

## 支持的公文类型

gongwen-draft 分两个层级支持公文和政务材料：

**法定公文（15 种，完整支持）**

覆盖《党政机关公文处理工作条例》规定的全部法定文种：决议、决定、命令（令）、公报、公告、通告、意见、通知、通报、报告、请示、批复、议案、函、纪要。这些文种均具备完整的起草、审核和 Word 导出能力。

**扩展政务材料（8 类）**

在日常政务工作中常见但不属于法定公文的材料类型：工作总结、工作方案、调研报告、讲话稿、汇报材料、简报、情况专报、回复函。

这 8 类材料均已配备专用结构模板，可以用于起草、改写、审核和 Word 草稿导出。

## 能做什么

**起草初稿。** 把你手头的会议记录、工作要点、调研材料交给它，它能按照对应文种的格式和行文规范，生成结构完整的公文初稿。通知、请示、报告、函、纪要、通报、批复、意见等均可。

**改写润色。** 已有草稿但行文不够规范时，它能调整语气、理顺逻辑、统一层级序号、纠正标点，使材料更符合公文写作习惯。

**格式审核。** 对已有材料进行自查，提示文种选用是否恰当、行文关系是否正确、事实表述是否完整、附件说明是否齐全、标点符号和层级序号是否合规等常见问题。

**素材整理。** 面对会议记录、政策摘录、调研材料、网页片段或多份本地素材时，先整理来源、事实、判断、待核实事项和起草风险，再进入写作。

**语言润色。** 对草稿进行规范化、压缩、正式化和风险降级处理，去掉口语化、空泛化、夸大化表达，同时保留原有事实和单位口径。

**Word 导出。** 在字体和字段条件满足时，将定稿导出为按公文格式排版的 Word 草稿文件，包括标题字体、正文字体、层级标题字体、页码等格式设置。

## 项目改进点

gongwen-draft 的目标是做一个更适合日常 Agent 工作流的公文写作 skill。相比常见静态提示词项目，它更重视以下几点：

**完整覆盖。** 同时覆盖 15 种法定公文和 8 类常见政务材料，并为每一类提供专用结构模板。

**先防风险。** 在起草前先判断文种、行文关系、事实边界、来源可靠性和权限边界，避免把未核实内容写成正式结论。

**可检查。** 内置文种覆盖校验、素材台账整理、语言风险检查、格式审核、标点检查和字体校验，尽量减少“规则写了但工具没跟上”的问题。

**可落地。** 支持 Word 草稿导出、字体检查、离线提示词包生成和 GitHub 自动校验，方便在不同 Agent 或内网环境中使用。

## 使用时要注意

公文写作涉及事实准确性和政策合规性，工具只能辅助，不能替代人工判断。以下几点请在使用中留意：

**事实和数据由你负责。** 重要的政策依据、统计数据、会议精神、领导讲话应由你提供或人工核实。工具不会主动编造这些内容，如果它不确定，会用占位标记提示你补充。

**涉密材料先脱敏。** 处分通报、人事任免、财政资金、行政处罚等高风险事项，请先脱敏并经过必要审核再使用。

**生成内容只是草稿。** 正式发文前仍需按单位流程完成审核、会签、签发和保密审查。工具生成的内容不具备任何行政效力。

**以单位现行制度为准。** 每个单位的公文模板和行文习惯可能有差异。如果你所在单位有自己的模板，应优先使用单位模板。

## 公文红线

以下底线不可突破：

不伪造、冒用或模拟真实机关的正式发文，包括机关名称、发文字号、签发人、印章和审批意见。

不编造政策依据、统计数据、会议结论或领导表述。

不把转载文章、培训材料或社区内容当作制度依据引用。

## Word 与字体

公文对字体有严格要求。gongwen-draft 已内置以下公文字体，安装后可以直接使用：

标题与发文机关标志使用方正小标宋简体，正文及三、四级标题使用仿宋\_GB2312，一级标题使用黑体，二级标题及签发人姓名使用楷体\_GB2312，页码使用宋体。

导出 Word 时，工具会先检查你的电脑是否已安装这些字体。如果缺少字体会停止导出并提示你，不会悄悄替换成其他字体。首次使用时，工具可以自动把内置字体安装到你的电脑中。

字体资产随项目提供用于公文草稿排版场景；使用、复制或二次分发时，请遵守字体来源方和所在单位的授权要求。
