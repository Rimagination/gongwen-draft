# gongwen-draft

![gongwen-draft Word 草稿预览](assets/readme-banner.png)

gongwen-draft 是一个面向 Codex 的中文公文与政务材料写作 skill，用于辅助起草、改写、审核、格式检查和 Word 草稿导出。

本项目的目标是在事实、文种、权限、格式、字体、标点和输出流程上尽量减少常见风险。它适合作为机关材料写作、内部初稿整理、格式自查和 Word 草稿生成的辅助工具，不替代单位现行模板、人工审核、审批流程或正式发文程序。

## 快速使用

安装：把本项目安装到 Codex 的 skills 目录，重启或刷新 Codex 后，在对话里直接说“使用 gongwen-draft”。

示例一：使用 gongwen-draft，帮我把下面的会议要点整理成一份通知初稿。

示例二：使用 gongwen-draft，把这份通知草稿整理成 Word 版，并检查字体、标点和附件格式。

## 能做什么

- 把会议记录、工作要点、调研材料整理成通知、请示、报告、函、通报、批复、意见、纪要等公文初稿。
- 审核已有材料中的文种、行文关系、事实表述、语气、层级序号、附件说明和标点问题。
- 提醒请示和报告混用、依据不明、事实未核、发文字号不当、英文标点混入等常见风险。
- 在字体可用且字段明确的前提下，导出符合常见公文格式习惯的 Word 草稿。

## 使用时要注意

- 重要事实、数据、政策依据、会议精神和领导讲话，应由用户提供或人工核实。
- 待核实内容应明确标注，不能写成确定结论。
- 涉密材料、内部审议材料、处分通报、人事任免、财政资金、行政处罚等高风险事项，应先脱敏并经过必要审核。
- 生成的内容只能作为草稿，正式发文前仍需按单位流程完成审核、会签、签发和保密审查。

## 公文红线

- 不伪造、冒用或模拟真实机关正式发文。
- 不生成未经授权的发文字号、签发人、印章、审批意见或正式版头。
- 不编造政策依据、统计数据、会议结论、领导表述或机关名称。
- 不把训练文章、转载材料或社区项目内容当作制度依据。

## Word 与字体

公文 Word 导出对字体较敏感。gongwen-draft 会优先检查所需字体，缺少字体时停止导出，不静默替换。

默认字体配置：

- 标题与发文机关标志：方正小标宋简体
- 正文及三、四级标题：仿宋_GB2312
- 一级标题：黑体
- 二级标题及签发人姓名：楷体_GB2312
- 页码：宋体

公开仓库默认不包含未确认再分发授权的字体文件。使用者应通过正式渠道或单位授权取得字体；公开发布或二次分发时，也应确认字体资产具备相应授权。
